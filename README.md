## 📝 Problema (Statement of Work - SDD)

El cliente ficticio **StoreFlow** requiere un **Sistema de Inventario básico pero profesional** que permita registrar, consultar y actualizar el stock de productos de manera confiable, sin depender de herramientas externas como hojas de cálculo.

Actualmente, la empresa enfrenta tres problemas críticos:

1. **Falta de visibilidad del inventario**  
   Los registros se mantienen de forma manual y dispersa, generando inconsistencias entre el inventario físico y el reportado.

2. **Errores en la actualización del stock**  
   Los movimientos de entrada y salida no siempre se registran a tiempo, dificultando evaluar disponibilidad real.

3. **Ausencia de trazabilidad histórica**  
   No existe historial de operaciones, imposibilitando auditorías, análisis de rotación o detección de anomalías.

---

### 🎯 Objetivo del Sistema

Desarrollar un sistema que permita:

- Registrar productos con atributos básicos (nombre, SKU, categoría, stock y precio).  
- Realizar operaciones de **entrada y salida de inventario**, manteniendo un historial auditable.  
- Consultar el estado actualizado del inventario a través de una API limpia, mantenible y bien documentada.  
- Operar sin costos adicionales (infraestructura local o gratuita).  
- Servir como base escalable, modular y apta para extensiones futuras.

---

### 🧩 Resultado Esperado

Un sistema **funcional, simple, profesional y containerizado** que solucione las necesidades del cliente y represente un proyecto sólido dentro del portafolio técnico del desarrollador.

# PROMPT: Functional Requirements (FR)

A partir del Feature Brief en `/specs/01-feature-brief.md`:

1. Genera la lista completa de Requerimientos Funcionales (FR).
2. Cada FR debe estar numerado y ser verificable.
3. Evita ambigüedades (no usar "fácil", "rápido", "bonito").
4. Relaciona cada FR con un caso de uso futuro.

Formato:

## Functional Requirements
- **FR-01:** …
- **FR-02:** …
…
